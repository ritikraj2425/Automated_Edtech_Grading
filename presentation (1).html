Automated Answer Sheet Grading

Upload a handwritten answer sheet, it detects each answer block, reads the text using OCR, and gives marks by comparing it with the teacher's answer.

Section 01

How I approached this

I didn't directly start writing code. First I read about what tools already exist for this kind of problem, tried a few things, and most of them didn't work the way I expected. Here's what I went through.

Tried — didn't work
Tesseract OCR alone

First I read about Tesseract. It reads text well, but the problem is it won't automatically detect "answers" as logical regions — it only detects text-level boxes like characters, words, lines, and paragraphs. So there's no way to know where Q1's answer ends and Q2's answer starts.

Read about — too complex for now
LayoutLM (Microsoft)

LayoutLM understands document layout along with text. But it needs fine-tuning on your specific document type and I didn't have labelled training data for answer sheets, so it wasn't practical to use right now.

Tried — needed too much training data
YOLO through LayoutParser

That's why I thought of using YOLO, which can automatically detect answer boxes as regions. Instead of training YOLO from scratch, I tried using a pre-trained YOLO model through LayoutParser. These models often work reasonably well on answer sheets because answers appear as large text blocks — but the pre-trained ones were trained on research papers and receipts, not exam sheets. Training a custom one needed hundreds of labelled images which I didn't have.

What I finally used
OpenCV horizontal projection

Then I realised I don't need a complex model for this. Answer blocks are separated by blank horizontal strips on the page. OpenCV can find these gaps just by counting ink pixels per row — rows with no ink are the gaps between answers. No training, no internet needed. It's not perfect but it works well enough for a standard answer sheet.

Section 02

What each tool does and why I used it

Block detection — using now
OpenCV — horizontal projection

Counts ink pixels row by row. Rows with no ink are gaps between answers. Groups the inky rows into blocks. No model, no training, runs instantly.

Block detection — future
Custom trained YOLO

If I collect and label enough answer sheet images, YOLO would give much more accurate detection — especially for messy or non-standard sheets.

Text reading — using now
Tesseract OCR

Once blocks are found, Tesseract reads the text inside each one. Free, works offline. I use PSM 6 mode which tells it to treat the crop as one uniform block of text.

Text reading — future
Google Vision API or TrOCR

Tesseract struggles with really messy handwriting. Google Vision handles it much better. TrOCR is a transformer model specifically built for handwritten text.

Scoring — using now
TF-IDF + Cosine Similarity

Converts student and teacher answers into vectors where important words get higher values. Then measures how similar those vectors are. No downloads, works in under a second. Main problem — it matches words, not meaning.

Scoring — future
Sentence-BERT

Understands meaning, not just word overlap. "Banks gave credit" and "loans were provided" would score high because they mean the same thing — TF-IDF would score them low.

Web server
Flask

Connects the browser to the Python code. When a teacher uploads an image, Flask receives it, runs OCR and marking, and sends results back. Simple to run locally and easy to deploy later.

Image preprocessing
Adaptive thresholding

Converts the photo to black and white. Needed because phone photos have uneven lighting. Adaptive thresholding adjusts per region so ink is detected cleanly everywhere.

Section 04

What about multiple questions on one sheet?

This is a real limitation right now. The system detects visual blocks top to bottom — it doesn't know which block is Q1 and which is Q2. Here's how it currently handles it.

How it works now
How to give input
Workaround

If a sheet has 3 answers, the system finds 3 blocks in order. It joins all 3 into one string and compares with the teacher's combined answer. So right now you get one score for the whole sheet, not one per question.

If a student did Q1 and Q3 well but Q2 badly, that gets averaged out and you can't see it separately.

For 3 questions, write all 3 answers together in the teacher answer box, one after the other:

1) Green revolution and white revolution were set up by the government. Farmers were given crop insurance. Kisan Credit Card was introduced. 2) Second answer here... 3) Third answer here...

The system combines all student blocks and compares with this combined teacher text. It's rough but works for checking overall topic coverage.

Right now the cleanest way is to crop the image to just one question's answer and upload that separately with just that question's model answer. Repeat for each question. Manual but gives accurate per-question scores.

The plan for the next version is to detect question labels like "Q1", "1)", "2." in the sheet using OCR and use those as automatic separators.

Section 05

What doesn't work well yet

Current problem
OCR accuracy on messy handwriting

Tesseract reads decent handwriting okay but struggles when it's really messy — "droughts" becomes "fleeds", "Kisan" becomes "Kison".

Current problem
No per-question score

All questions on a sheet get one combined score. Can't say Q1 = 8/10 and Q2 = 4/10 separately right now.

Current problem
TF-IDF matches words, not meaning

"Loans were given by banks" and "credit was provided by financial institutions" mean the same thing but score low because the actual words don't overlap.

Current problem
No PDF support

Only accepts image files. PDF answer sheets can't be uploaded directly yet.

Next version
Auto question detection

Detect "Q1", "1)", "2." labels in the sheet and automatically split and score each question separately.

Next version
PDF upload

Convert PDF pages to images before processing so teachers can upload scanned PDFs directly.

Later
Better similarity — Sentence-BERT

Replace TF-IDF with a model that understands meaning so paraphrased answers get fair scores.

Later
Full class report

Upload sheets for all students, get a report with everyone's scores, class average, and which questions were most commonly wrong.
